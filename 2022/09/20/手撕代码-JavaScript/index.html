instanceof

MDN:instanceof 运算符用于检测构造函数的 prototype 属性是否出现在某个实例对象的原型链上。

实现思路：
如果a instanceof B，首先 a 必须要是个对象，其次 B 也必须是一个合法的[构造]函数。在这两个条件都满足的情况下：判断 B 的 prototype 属性指向的原型对象（ B.prototype ）是否在实例对象 a 的原型链上。如果在，则返回 true；如果不在，则返回 false。
专业点说：只要遍历实例对象的原型链，挨个往上查找看是否有与构造函数的 prototype 相等的原型，直到最顶层 Object 还找不到，那么就返回 false，否则结果就是 true.
再直白一点 instanceof就是一个查找原型链的过程。

call

MDN:call() 方法使用一个指定的 this 值和单独给出的一个或多个参数来调用一个函数。
function.call(thisArg, arg1, arg2, …)

thisArg：
在 function 函数运行时使用的 this 值。请注意，this 可能不是该方法看到的实际值：如果这个函数处于非严格模式下，则指定为 null 或 undefined 时会自动替换为指向全局对象(window)，原始值会被包装。
实现思路：
核心点是为传进来的对象 context 添加 fn 这个函数属性，然后 context 就可以执行 fn 这个函数，也就是改变了 fn 的 this 指向
